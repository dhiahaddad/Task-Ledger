# Security policy

## Supported versions

Until the first stable release, security fixes are made on the latest development
version only.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Contact the maintainer
privately with a description, reproduction steps, affected versions, and any suggested
mitigation. Allow reasonable time to investigate and prepare a coordinated fix before
public disclosure.

Never include real secrets, private task data, or personally identifying information
in a report. Use synthetic examples.
